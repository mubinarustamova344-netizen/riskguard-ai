2. FRAMEWORK VA TEXNOLOGIYALAR / FRAMEWORKS & TECHNOLOGIES
--------------------------------------------------------------------------------

  BACKEND
  -------
  Flask 3.0           Web framework and HTTP routing
  Flask-Login         Session-based authentication (8-hour timeout)
  Flask-WTF           CSRF protection and form validation
  Flask-Limiter       Rate limiting (10/min login, 60/min API)
  SQLAlchemy          ORM for SQLite database access
  Werkzeug            Password hashing (bcrypt) and security utilities
  python-dotenv       Environment variable management (.env file)
  ReportLab           PDF report generation (in-memory, no temp files)

  MACHINE LEARNING
  ----------------
  scikit-learn 1.3    ML pipeline: preprocessing, training, evaluation
  XGBoost 3.2         Production model (85.22% accuracy, ROC-AUC 0.9738)
  pandas              Data processing and CSV handling
  numpy               Numerical operations
  joblib              Model serialisation (.pkl files)

3. DATABASE TURI / DATABASE
--------------------------------------------------------------------------------

  Type       : SQLite (file-based, zero-configuration relational database)
  ORM        : SQLAlchemy (Flask-SQLAlchemy)
  File       : assessments.db  [auto-created on first run]
  Location   : Source_Code/ root directory

  Table: assessments
    id, timestamp, driver_age, driving_experience, annual_mileage,
    vehicle_age, previous_accidents, traffic_violations, night_driving_pct,
    credit_score, vehicle_type, primary_location, marital_status, gender,
    risk_score, risk_category, premium_multiplier, claim_probability,
    recommendations, factor_contributions

  Training Dataset:
    30,000 synthetic records | 18 features (12 input + 6 engineered)
    Generated by: data/generate_data.py
    Risk split: Low 35% | Medium 35% | High 20% | Very High 10%

  For production: migrate to PostgreSQL
    pip install psycopg2-binary
    Update SQLALCHEMY_DATABASE_URI in app.py

4. LOYIHANI ISHGA TUSHIRISH / HOW TO RUN THE PROJECT
--------------------------------------------------------------------------------

  SYSTEM REQUIREMENTS
  -------------------
  Operating System : Windows 10/11 (or macOS 12+ / Ubuntu 20+)
  Python Version   : 3.11 or higher  (developed on Python 3.13)
  RAM              : Minimum 4 GB  (8 GB recommended for model training)
  Disk Space       : ~300 MB (including virtual environment)
  Browser          : Chrome 90+, Firefox 88+, Edge 90+

  ......................................................................

  STEP 1 — Install Python
  -----------------------
  Download Python 3.13 from:  https://python.org/downloads
  During installation, CHECK "Add Python to PATH"

  Verify:
    python --version
    pip --version

  ......................................................................

  STEP 2 — Install dependencies
  ------------------------------
  Open terminal (Command Prompt or PowerShell) in the Source_Code/ folder:

    pip install -r requirements.txt
    pip install reportlab werkzeug flask-wtf

  ......................................................................

  STEP 3 — Create the .env file
  ------------------------------
  In the Source_Code/ folder, create a file named exactly:  .env
  (note the dot at the start — no .txt extension)

  Paste this content and fill in your values:

    SECRET_KEY=any-random-secret-string-here-change-this
    ADMIN_USERNAME=admin
    ADMIN_PASSWORD=yourpassword
    ANTHROPIC_API_KEY=sk-ant-...    [optional - enables Claude AI chatbot]
    GEMINI_API_KEY=AIza...          [optional - enables Gemini AI chatbot]
    EMAIL_USER=                     [optional - Gmail for notifications]
    EMAIL_PASS=                     [optional - Gmail app password]

  Note: The app works WITHOUT API keys. The chatbot uses regex fallback.

  ......................................................................

  STEP 4 — First-time setup (REQUIRED - run once only)
  -----------------------------------------------------
  This generates 30,000 training records and trains the XGBoost model.
  Takes approximately 30-60 seconds:

    python setup.py

  Expected output:
    [1/3] Generating 30,000 synthetic records...
    [2/3] Training 6 ML models with GridSearchCV...
    [3/3] Best model: XGBoost — Accuracy: 85.22%, ROC-AUC: 0.9738
    Setup complete. Run: python app.py

  This creates:
    model/risk_model.pkl      (trained XGBoost pipeline, ~5 MB)
    model/model_metrics.pkl   (evaluation results for all 6 models)

  ......................................................................

  STEP 5 — Start the application
  --------------------------------
  Option A — Windows easy launcher (recommended):
    Double-click:  run.bat

  Option B — Manual:
    python app.py

  The app auto-selects a free port starting from 5000.
  Open your browser at:   http://127.0.0.1:5000

  ......................................................................

  STEP 6 — Log in
  ----------------
  Username: admin         (or your ADMIN_USERNAME from .env)
  Password: yourpassword  (or your ADMIN_PASSWORD from .env)

  ......................................................................

  TROUBLESHOOTING
  ---------------
  "ModuleNotFoundError: No module named 'flask'"
    -> Run:  pip install -r requirements.txt

  "ModuleNotFoundError: No module named 'reportlab'"
    -> Run:  pip install reportlab

  "Model not found - run setup.py"
    -> Run:  python setup.py

  "Address already in use" / port conflict
    -> The app auto-tries ports 5000-5010. Or set PORT in .env.

  Chatbot responds in English to Uzbek messages
    -> This is expected without an API key. The regex FAQ covers
       common Uzbek queries. Add ANTHROPIC_API_KEY for full AI.

6. API ENDPOINTS (for developers)
--------------------------------------------------------------------------------

  POST  /api/assess         Perform a risk assessment (JSON input)
  GET   /api/stats          Dashboard statistics summary
  GET   /api/history        All assessment records (JSON)
  GET   /api/export-csv     Export all records as CSV download
  POST  /api/compare        Compare two driver profiles
  POST  /api/batch-upload   Bulk assess from CSV file
  POST  /api/chat           Chatbot non-streaming response
  GET   /api/chat/stream    Chatbot streaming response (SSE)
  POST  /api/chat/reset     Clear chatbot conversation history
  GET   /api/model-metrics  ML model evaluation results

  Full interactive documentation: http://127.0.0.1:5000/api-docs

7. SECURITY FEATURES
--------------------------------------------------------------------------------

  CSRF Protection     All POST forms protected (Flask-WTF tokens)
  Rate Limiting       Login: 10/min | API: 60/min (Flask-Limiter)
  Password Hashing    bcrypt with work factor 12 (Werkzeug)
  Session Security    HTTPOnly + SameSite=Lax cookies; 8h timeout
  Security Headers    X-Frame-Options: DENY
                      X-Content-Type-Options: nosniff
                      Referrer-Policy: strict-origin-when-cross-origin
  Input Validation    All 12 input fields validated server-side
  SQL Safety          Parameterised queries via SQLAlchemy ORM
